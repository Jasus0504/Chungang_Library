2. [PHP]

회원가입 및 Login 페이지는 필수 (로그인 후, 회원 전용 페이지와 비회원 페이지 구별)
나머지는 주제와 관련된 페이지로 구성하되, 1~3번의 조건/요구사항을 만족하는 페이지로 구성한다. 

기능(주요 기능)
-이미 대여된 책은 대여가 불가능, 도서리스트에서 대여불가능이라 뜨지만, 대여페이지에서 해당 도서 입력 후 도서번호를 입력하고 대여하기 버튼을 누르면 이미 대여된 책이라는 alert 발생. 

-도서리스트에서 바로 대여하는 방식을 하지 않은 이유는 실제 도서관에서 책을 직접 손으로 가져오지 않은 상태에서 대여를 하면 혼선이 발생할 수 있으며 실제 도서관에서도 해당책을 바코드로 찍고 빌리는 형태임을 감안하여 책 번호를 입력하여 빌리는 것으로 대체 (테스트를 위해서 검색할 때 책 번호 표시)

-책을 빌릴 수 있는 것은 최대 3권으로 그 이상은 반납후 대여 가능하게 설계

-빌린 책을 반납할때마다 도서카운트를 늘려서 가장 많이 빌린 책을 메인화면에서 확인 가능, 반납할때 책 리뷰를 작성하면 책 상세페이지에서 리뷰랑 평균평점을 관람 가능

-같은 책이라 할지라도 2권이상 존재할 수 있기에 설계(사실 설계 측면에서 생각하지 못한것이라,최적의 방법으로 구현하지는 못함) 예를들어 아기돼지 삼형제가 같은 종류지만 이용자가 많이 빌려가는 책이라 같은 종류의 책을 2권 보유할 수 있음. 또는 버전이 다르지만 같은 책을 보유할 수도 있음.  이때 한권을 빌려가면 도서 현황에서 1/2로 표시하고 리뷰와 평점은 하나로 통합

-방문자, 로그인 성공한 유저, 관리자 페이지를 별도로 제공

-관리자만 회원정보 및 도서 정보 삭제 및 수정 가능

-책 검색은 도서/작가 이름으로 가능

4.보안 

1. SQL Injection 방지
	- Prepared Statement(준비된 문장)와 Parameterized Query(매개변수화된 쿼리)를 사용, 쿼리문을 고정함으로써 SQL Injection방지 
	-(book/checkout_search에 사용)
2. Cross-Site Scripting (XSS) 방지
	-htmlspecialchars 함수를 통해 HTML 특수 문자를 HTML 엔티티로 변환하여 브라우저가 이를 HTML이나 JavaScript 코드로 해석하지 않고 단순한 텍스트로 처리하도록 설계
	-(book/usb_selectAll에 사용)
3.세션 제한시간
	-로그인 후 만약 움직임이 30분 이상 없으면 세션을 삭제하도록 설정했습니다
	-(session.php 에 사용)
